This component creates similar functionality as Total.js Actions. ## Payload ```js { ip: String, // optional, IP address url: String, // optional language: String, // optional ua: String, // optional, parsed user-agent user: Object, // optional headers: { key: String }, // optional, request headers cookies: { key: String }, // optional, cookies payload: {}, // optional, request payload query: { key: String }, // optional params: { key: String }, // optional files: [{ type: String, filename: String, path: String, ext: String, size: Number, width: Number, height: Number }] // optional } ``` ## Custom scripting - supports `await` - supports Total.js global functions __Properties__: - `$.model {Object}` or `model` parsed payload - `$.query {Object}` parsed URL query arguments - `$.params {Object}` parsed URL params - `$.cookies {Object}` parsed cookies - `$.files [Object]` uploaded files - `$.user {Object}` user session (optional) - `$.headers {Object}` request headers - `$.ip {String}` IP address (optional) - `$.language {String}` optional - `$.ua {String}` parsed user agent (optional) - `$.url {String}` URL endpoint - `$.message {Object}` Flow message instance - `$.request {Object}` Flow message payload data __Methods__: - `$.invalid(error)` responds with an error and ends the action - `$.success([value])` responds with `{ success: true, value }` object and ends the action - `$.callback(value)` responds with `value` object and ends the action - `$.cancel()` ends the action without response - `$.audit(message, [type])` writes audit ## Supported types - `string` - `number` - `boolean` - `date` - `object` - `email` - `phone` - `zip` - `base64` - `url` - `datauri` - `json` - `lower` - `upper` - `capitalize` all words - `capitalize2` only first word - `name` makes a name (capitalizes words and remove special characters) - `zip` - `uid` - `guid` - `color` - `icon` - `search` prepares string to a search string - `safestring` checks XSS and SQL Injections - `smallint` - `tinyint` - `{A|B|C}` enum type - `[type]` means array - `{ name:String, age:Number }` means nested object - `[ name:String, age:Number ]` means array with an object - `*something:type` is `required` field
